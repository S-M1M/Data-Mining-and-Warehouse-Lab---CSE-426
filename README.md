# 💼 JobFinder: Specialized Search Engine for Job Listings

## 📌 Project Overview
JobFinder is a custom-built, specialized search engine designed exclusively for retrieving and ranking job listings. Built entirely from scratch without the use of pre-built search libraries (like Elasticsearch), this project demonstrates core information retrieval concepts, including text preprocessing, inverted indexing, Boolean query logic, and TF-IDF ranking.

## ✨ Features
* **Custom Inverted Index:** Maps extracted terms to document IDs and tracks term frequencies.
* **Boolean Retrieval:** Supports complex multi-word queries using `AND` / `OR` logic (e.g., `Python AND Remote`).
* **TF-IDF Ranking:** Dynamically ranks search results based on Term Frequency-Inverse Document Frequency.
* **Web Interface:** A clean, modern GUI built with Streamlit, deployed via Pinggy directly from Google Colab.

## 🛠️ Technology Stack
* **Language:** Python
* **Data Manipulation:** Pandas, NumPy
* **NLP & Preprocessing:** NLTK (Tokenization, Stop-word removal)
* **Web Framework:** Streamlit
* **Deployment:** Pinggy (for Colab tunneling)

## 📂 Repository Structure
* `JobFinder_Notebook.ipynb`: The complete Google Colab notebook containing data downloading, indexing, and UI launch code.
* `app.py`: The Streamlit web application script generated by the notebook.
* `search_engine_data.pkl`: The serialized inverted index and document frequencies (generated during execution).
* `README.md`: Project documentation.

## 🚀 How to Run
1. Open the `JobFinder_Notebook.ipynb` in Google Colab.
2. Run **Cells 1 through 8** to download the dataset, process the text, and build the inverted index.
3. Run **Cells 9A, 9B, and 9C** to serialize the data, generate the Streamlit app, and launch the Pinggy tunnel.
4. Click the generated `pinggy.link` URL in the output to access the web interface. No password is required.
